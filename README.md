## ✨ Key Features

-   **Real-Time Updates**: Uses Server-Sent Events (SSE) to push new clips to the dashboard instantly as they are recorded.
-   **Smart Thumbnails**: 
    -   Automatically uses high-res ML snapshots for person-detected clips.
    -   Extracts frames dynamically in the browser for standard motion clips.
-   **Highlights Toggle**: Toggle person-detection bounding boxes on/off with a single switch.
-   **Multi-Day Calendar**: Select multiple days to view merged event history.
-   **Activity Visualization**: Hourly histogram showing motion vs. person intensity.
-   **Persistent Tagging**: Add custom tags (family, delivery, etc.) saved to a `tags.json` sidecar file.
-   **Smart Icons**: Automatic "Heart/Family" icon for clips tagged as family.
-   **Flexible Density**: Toggle between high-density (small) and featured (large) card views.
-   **Premium Aesthetics**: Modern glassmorphic dark mode with a mobile-responsive drawer-based navigation.

## 🚀 Installation & Setup

### 🪟 Windows Setup

1.  **Prerequisites**: Install [Node.js](https://nodejs.org/) (LTS recommended).
2.  **Download**: Clone or download this repository to your computer.
3.  **Install Dependencies**:
    ```powershell
    cd watcher_web
    npm install
    ```
4.  **Configure Directory**:
    Set your video directory path as an environment variable:
    ```powershell
    $env:ARLO_DIR = "C:\Users\YourUser\Videos\Arlo"
    ```
5.  **Start the Server**:
    ```powershell
    node server.js
    ```
6.  **Access**: Open `http://localhost:3456` in your browser.

---

### 🐧 Linux (Ubuntu) Setup

I have provided automated scripts for a production-ready deployment.

1.  **Prerequisites**: Ensure `git` and `curl` are installed.
2.  **Run Installer**:
    ```bash
    chmod +x install.sh
    ./install.sh
    ```
    *This will install NVM, Node.js, dependencies, and generate a systemd service template.*
3.  **Configure Environment**:
    Edit the `.env` file created in the project root:
    ```bash
    nano .env
    # Set ARLO_DIR=/path/to/your/videos
    ```
4.  **Enable System Service**:
    ```bash
    sudo cp watcher-web.service /etc/systemd/system/
    sudo systemctl daemon-reload
    sudo systemctl enable watcher-web
    sudo systemctl start watcher-web
    ```
5.  **Updates**:
    To update the app in the future, simply run:
    ```bash
    ./update.sh
    ```

## 📂 File Structure

-   `server.js`: Express server with Chokidar file watching.
-   `public/`: Frontend assets (Vanilla HTML/CSS/JS).
-   `processed_files.jsonl`: Metadata file generated by your detection service.
-   `tags.json`: Persistent tag storage.
